## FILTERABLE PROMPT STRUCTURE

Every prompt in a successful response MUST include:

1. **prompt** (string): The fully resolved question with specific values
2. **sudoPrompt** (string): Template with placeholders for filterable values
3. **filters** (object): Key-value pairs of filter names and their options

### Filter Naming Conventions:

**Reserved filter keys:**
- `calendar`: Always array of two dates [start_date, end_date] in format "YYYY-MM-DD"
  - Placeholder in sudoPrompt: `<calendar_start>` and `<calendar_end>`
  
**Common filter keys:**
- `period`: Time aggregation (daily, weekly, monthly, quarterly, yearly)
- `channel`: Marketing channel (Paid Search, Social Media, Display, Email, etc.)
- `product_category`: Product groupings
- `region`: Geographic segments
- `customer_segment`: Customer groupings
- `platform`: Advertising platforms
- `comparison_period`: For comparison filters (previous_period, previous_year, custom)

**Placeholder format in sudoPrompt:**
- Use angle brackets: `<filter_key>`
- For calendar: Use `<calendar_start>` and `<calendar_end>` separately
- Example: `"Show <metric> for <channel> from <calendar_start> to <calendar_end>"`

### Filter Value Guidelines:

**For calendar filters:**
```json
"filters": {
  "calendar": ["2026-01-01", "2026-01-31"]
}
```

**For enumerated options:**
```json
"filters": {
  "period": ["daily", "weekly", "monthly", "quarterly", "yearly"],
  "channel": ["Paid Search", "Social Media", "Display", "Email", "All Channels"]
}
```

**Multiple filters in one prompt:**
```json
{
  "prompt": "What is the total revenue for Electronics category from 2026-01-01 to 2026-01-31 aggregated monthly?",
  "sudoPrompt": "What is the total revenue for <product_category> from <calendar_start> to <calendar_end> aggregated <period>?",
  "filters": {
    "calendar": ["2026-01-01", "2026-01-31"],
    "product_category": ["Electronics", "Clothing", "Home & Garden", "Sports", "All Categories"],
    "period": ["daily", "weekly", "monthly", "quarterly", "yearly"]
  }
}
```

## STEP 4: FILTER IDENTIFICATION

For each prompt, identify filterable dimensions based on the question type.

### Standard Filters by Question Type:

**Time-based metrics:**
- Always include: `calendar` filter
- Often include: `period` filter (daily, weekly, monthly, quarterly, yearly)

**Marketing metrics:**

- `platform`: ["Google Ads", "Facebook", "Instagram", "LinkedIn", "TikTok", "All Platforms"]

**Sales/Revenue metrics:**
- `calendar`: date range
- `period`: aggregation level
- `channel`: ["Online", "Retail", "Wholesale", "Marketplace", "All Channels"]
- `region`: ["North America", "Europe", "Asia", "Other", "All Regions"]

**Product metrics:**
- `calendar`: date range
- `product_category`: ["Electronics", "Clothing", "Home & Garden", "Sports", "Beauty", "All Categories"]
- `brand`: ["Brand A", "Brand B", "Brand C", "All Brands"]
- `price_range`: ["Under $50", "$50-$100", "$100-$200", "Over $200", "All Prices"]

**Customer metrics:**
- `calendar`: date range
- `customer_segment`: ["New", "Repeat", "VIP", "At Risk", "All Customers"]
- `region`: geographic filters
- `acquisition_channel`: ["Organic", "Paid", "Referral", "Social", "All Channels"]

### Filter Option Guidelines:

1. **Always include "All [Category]"** as the last option for non-calendar filters
2. **Limit options to 3-7 choices** (excluding "All")
3. **Use business-friendly labels**, not technical field names
4. **Order options logically**: Most common first, "All" last
5. **Calendar filter always has exactly 2 dates**: [start, end]

## STEP 6: DIMENSION SAFETY

**SAFE dimensions** (always filterable):
- `calendar`: dates
- `period`: time aggregation
- Explicitly mentioned dimensions in user query

**UNSAFE dimensions** (avoid unless mentioned):
- `campaign`: requires campaign-level data
- `product_sku`: requires SKU-level tracking
- `customer_id`: requires individual customer data
- `region`: requires geographic tagging
- `brand`: requires brand attribution

**Safety Rules:**
1. Only include filters for dimensions mentioned in query OR universally available
2. If uncertain about dimension availability, exclude it
3. Always provide "All [Category]" option as safe default
4. Never generate prompts requiring joins across tables
