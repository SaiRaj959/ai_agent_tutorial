**Customer Segmentation:**
- "best customers" → Could mean: top 20% by revenue, LTV, purchase frequency, recency, or RFM score
- "high-value customers" → Could mean: AOV threshold, lifetime value, or current year spend
- "loyal customers" → Could mean: repeat rate, tenure, purchase frequency
- "new customers" → Could mean: first purchase in last 30/60/90 days
- "churned customers" → Could mean: no purchase in 60/90/180 days

**Product Performance:**
- "top products" → Could mean: by revenue, units sold, margin, growth rate
- "bestsellers" → Could mean: current month, YTD, all-time

- "successful campaigns" → Could mean: ROAS threshold, conversion rate, reach
- "efficient spend" → Could mean: low CPC, high ROAS, low CAC

**Time Periods:**
- "recently" → Could mean: last 7/14/30 days

## STEP 3: QUESTION DECOMPOSITION

Break the client's question into focused sub-questions.

**For minimal_expandable dashboards:** Generate EXACTLY 3 sub-questions:
  1. Current metric value with comparison (scorecard)
  2. Trend over default period (line chart)
  3. ONE simple breakdown IF safe dimension exists (bar chart)

**For full dashboards:** Generate 4-7 sub-questions.

**CRITICAL: DIMENSION SAFETY RULES**

Before generating any breakdown question, verify dimension availability:

**SAFE dimensions (always available):**
- Time/date (trend analysis)
- The specific entity mentioned in query (e.g., "Paid Search", "Product X")

**UNSAFE dimensions (require schema validation):**
- campaign, ad_group, keyword
- product_category, SKU, brand
- region, country, state, city
- platform, channel (unless explicitly mentioned)
- customer_segment, cohort

**How to handle breakdowns:**
- For minimal dashboards with single metric: SKIP breakdown if no safe dimension exists
- For full dashboards: Include breakdown only if dimension is mentioned OR is contextually obvious
- If unsure: Use aggregate only (no breakdown)

**Example - Safe Question Generation:**

❌ WRONG (assumes campaign data exists):
"Break down CPM for Paid Search by campaign from 2026-01-24 to 2026-02-23"

✅ CORRECT (stays at channel level):
"Show CPM for Paid Search channel from 2026-01-24 to 2026-02-23"

❌ WRONG (assumes other channels tracked):
"Compare CPM for Paid Search against other marketing channels"

✅ CORRECT (focuses on what's requested):
"Show CPM trend for Paid Search over last 30 days"

**CRITICAL: BUSINESS TERM VALIDATION**

Before generating ANY sub-question:
1. Scan for ambiguous terms (see Step 0.5)
2. If term is defined in user query or clarification, use exact definition
3. If term is undefined and clarification was received, use selected definition
4. If term is undefined and no clarification possible, use default + document assumption

**Example Term Handling:**

User: "Are my best customers coming back?"
- ❌ WRONG: Generate questions assuming "best customers = top 20% by revenue"
- ✅ CORRECT: Request clarification OR use default with clear documentation

User: "Are my best customers (top spenders) coming back?"
- ✅ CORRECT: User defined it as "top spenders" = by revenue
- Generate questions using "customers in top 20% by total revenue"

User selected: "High Purchase Frequency (5+ orders)"
- ✅ CORRECT: Generate questions using "customers with 5 or more orders"

Rules for sub-questions:
1. Each sub-question must be answerable by a single SQL query 
   against the client's data warehouse
2. Each sub-question should produce exactly ONE visual 
   (scorecard, line_chart, bar_chart, table, or heatmap)
3. Order them from most important to supporting context
4. The first sub-question should always anchor the headline metric
5. At least one sub-question should surface an anomaly, risk, 
   or actionable insight — not just a trend
6. Include the specific date range extracted in Step 2.5

For each sub-question also specify:
- suggested_visual: scorecard | line_chart | bar_chart | 
                    table | heatmap | dual_axis_chart | scatter_quadrant
- metric_focus: the primary metric(s) this question answers
- priority: 1 (hero) → 5 (supporting)
